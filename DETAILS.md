# Производительность способов регистрации правил

Пакет поддерживает три способа задать SEO-правило. По чистой загрузке — от самого эффективного к менее:

## 1. Инлайн-вызов `Xseo::create([Class::class, 'method'], ...)`

Минимальные накладные расходы. Класс автозагружается и инстанцируется ровно в момент вызова, никакого промежуточного реестра нет вообще — ни обращения к `config('xseo.rules')`, ни требования файла.

Единственный минус: если один и тот же handler вызвать несколько раз за запрос, он резолвится через контейнер заново каждый раз (в `XseoManager::normalizeRule()` для array/Closure мемоизации нет — незачем, нет стабильного имени, под которым кэшировать).

## 2. `config('xseo.rules')` (class/array-правила)

Практически то же самое по стоимости для одного вызова: тоже ленивое разрешение конкретного имени через `resolveRule()`, плюс мемоизация в `$this->rules[$name]` (см. `src/XseoManager.php`, методы `resolveRule()`/`makeRuleClosure()`).

Разница с инлайном — один lookup в массиве конфига (микросекунды, не считово). Выигрывает у инлайна, если то же самое правило вызывается несколько раз за запрос — второй вызов берёт уже готовое закешированное замыкание.

## 3. `config('xseo.files')` (closures в rules.php) — самый затратный

При первом резолве `xseo` за запрос (`XseoServiceProvider::boot()`, `resolving('xseo', ...)`) весь файл `require`-ится целиком — регистрируются **все** `Xseo::rule(...)` из файла, даже те, что в этом запросе не используются. Стоимость растёт с числом правил в файле, а не с тем, сколько реально нужно.

## Итог

Варианты 1 и 2 по факту равны на практике (разница на уровне погрешности), оба на порядок эффективнее варианта 3, потому что грузят/резолвят ровно то, что используется, а не всё разом.

В pilar.loc в `resources/xseo/rules.php` сейчас всего два правила (`default` + `home`) — там разница копеечная, но при росте числа правил `files`-механизм будет тормозить сильнее всего.

**Когда что использовать:**
- Инлайн `[Class::class, 'method']` / `Closure` — правило специфично для одного места вызова, регистрировать его где-то отдельно незачем.
- `config('xseo.rules')` — правило переиспользуется в нескольких местах под одним именем, или важна совместимость с `config:cache`.
- `config('xseo.files')` — годится для быстрого прототипирования (просто дописал `Xseo::rule(...)` в файл), но не масштабируется так же хорошо, как первые два варианта.
